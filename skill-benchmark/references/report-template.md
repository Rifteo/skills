Do not add extra sections. Do not skip sections.

## Error output format

If any step fails (unreadable file, missing reference doc, malformed frontmatter),
stop immediately and output a single line in this exact form:

```
[skill-benchmark error] <what failed> — <what the user should do>
```

Examples:
- `[skill-benchmark error] references/rubric.md not found — ensure the full skill-benchmark package is installed`
- `[skill-benchmark error] Frontmatter missing 'name' field — add a name: key to the skill's YAML front matter`

Do not attempt to continue past an error.

---

## Notes for the agent filling this template

- **Scores must add up.** Double-check that dimension scores sum to the total.
- **Be specific in fixes.** "Improve the description" is not a fix. "Add trigger
  phrases for 'audit my skill', 'check compatibility', 'score this SKILL.md' to
  the description field" is a fix.
- **Estimated pts are honest, not encouraging.** If fixing an issue realistically
  adds 2 pts, say +2, not +8.
- **Compatibility count must equal 54.** Every agent in the registry must appear
  in exactly one of the three groups.
- **Do not invent problems.** If a dimension scores 20/20, say so and move on.
